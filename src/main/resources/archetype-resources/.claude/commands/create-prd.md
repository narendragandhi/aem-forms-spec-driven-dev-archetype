# Create PRD Workflow

You are initiating the **Create PRD** workflow from BMAD v6.

## Workflow Steps

### Step 1: Project Context
Gather the following information:
- Project name and description
- Target users and personas
- Business objectives and success metrics

### Step 2: Vision & Differentiators (v6 2b)
Define:
- Product vision statement
- Key differentiators from competitors
- Unique value proposition

### Step 3: Executive Summary (v6 2c)
Create a concise executive summary covering:
- Problem statement
- Proposed solution
- Expected outcomes
- High-level timeline

### Step 4: Requirements
Document functional and non-functional requirements:
- User stories with acceptance criteria
- Technical constraints
- Integration requirements
- Performance requirements

### Step 5: Output
Generate the PRD document at:
`bmad/01-Business-Discovery/requirements.md`

## Artifacts to Create
1. `bmad/01-Business-Discovery/requirements.md` - Main PRD
2. `bmad/01-Business-Discovery/user-stories.md` - Detailed user stories
